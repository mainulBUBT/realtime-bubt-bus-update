## Key Technical Domains Identified

1. **Core Architecture**
   - Laravel 11 backend with Vue 3 frontend
   - Laravel Reverb (WebSocket) for real-time updates
   - Capacitor for mobile apps (iOS/Android)
   - Multi-trip tracking with `BusTrip` model

2. **Database Schema**
   - Core: `users`, `buses`, `bus_routes`, `bus_stops`, `bus_schedules`
   - Tracking: `bus_trips`, `bus_active_users`, `user_locations`, `bus_locations`
   - Config: `system_settings`, `schedule_periods`

3. **API Endpoints**
   - Public: `GET /api/buses`, `GET /api/buses/{id}`
   - Protected: `POST /api/confirm-bus`, `POST /api/leave-bus`, `POST /api/save-location`, `GET /api/active-bus`

4. **Scheduled Jobs**
   - `DailyCleanupJob` - Runs at 12:05 AM, deletes locations > 30 days
   - `CompleteExpiredTripsJob` - Runs every 5 minutes
   - `CleanupInactiveUsersJob` - Runs every 2 minutes

5. **System Settings**
   - `min_active_users`: 2
   - `inactive_user_timeout`: 120 seconds
   - `location_max_age`: 120 seconds
   - `route_proximity_threshold`: 100 meters
   - `auto_complete_trips_after_minutes`: 10
   - `trip_duration_buffer_hours`: 4

## ROOT CAUSE ANALYSIS

The **documentation is significantly outdated**. It describes:
1. An older/mifferent API structure (public bus endpoints)
2. Different table naming conventions (`bus_*` prefix)
3. Different models (BusTrip vs Trip, UserLocation vs Location)
4. Jobs that were later removed
5. Features that were refactored or consolidated

The codebase was refactored but documentation was never updated.

## DETAILED FINDINGS

### Issue #1: API Routes Completely Mismatch

**DOCUMENTED Routes** (in docs/deployment.md, docs/features/*.md):
```
GET    /api/buses           → List all buses (public)
GET    /api/buses/{id}      → Get bus details (public)
POST   /api/confirm-bus     → Student confirms on bus
POST   /api/leave-bus       → Student leaves bus
POST   /api/save-location   → Submit GPS location
GET    /api/active-bus      → Get user's active bus
GET    /api/route/distance  → OSRM road distance
```

**ACTUAL Routes** (in routes/api.php):
```
# Public
POST   /api/auth/login
POST   /api/auth/register
GET    /api/settings

# Auth (protected)
POST   /api/auth/logout
GET    /api/auth/me
PATCH  /api/auth/profile
PATCH  /api/auth/password

# Driver (role:driver)
GET    /api/driver/buses
GET    /api/driver/routes
POST   /api/driver/trips/start
POST   /api/driver/trips/{trip}/end
GET    /api/driver/trips/current
GET    /api/driver/trips/history
POST   /api/driver/location
POST   /api/driver/location/batch

# Student (role:student)
GET    /api/student/routes
GET    /api/student/routes/{id}
GET    /api/student/trips/active
GET    /api/student/trips/{tripId}/locations
GET    /api/student/trips/{tripId}/latest-location
GET    /api/student/schedules
POST   /api/student/fcm-token
GET    /api/student/notifications
...

# Admin (role:admin)
GET/POST/PUT/DELETE /api/admin/buses
GET/POST/PUT/DELETE /api/admin/routes
GET/POST/PUT/DELETE /api/admin/schedules
```

**Impact:** Apps expecting documented public endpoints will fail. No public bus list/detail endpoints exist.
